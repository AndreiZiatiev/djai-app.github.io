Getting Started (Quickstart)

The easiest way to get started with the Apicurio Studio is to download the quickstart (based on Tomcat 8) and fire it up! All you need is to have Java (version 8) installed, and you're ready to rock.

Download & Install

  1. Apicurio Studio {{ latestRelease.tag_name }}
  2. Unpack the Apicurio Studio Quickstart zip

Configure

  1. In order to log in to the quickstart, you will need to have a GitHub account and you will need to register an OAuth application in GitHub.
  2. Make sure you use the following important settings when you register your OAuth app:
    Homepage URL
    : https://localhost:8443/studio
    Authorization URL
    : https://localhost:8443/studio/callback
  3. Create environment variables for your GitHub OAuth app's ClientID and ClientSecret using GITHUB_AUTH_CLIENT_ID and GITHUB_AUTH_CLIENT_SECRET, respectively.

Run

  1. Start WildFly
  2. Point your browser at Apicurio Studio and authenticate with your GitHub account!

Or try this (after registering an OAuth app)...

mkdir ~/apicurio-studio-{{ site.version }}
cd ~/apicurio-studio-{{ site.version }}
curl -L {{ theasset.browser_download_url }} -o {{ theasset.name }}
unzip {{ theasset.name }}
cd apicurio-studio-{{ site.version }}
GITHUB_AUTH_CLIENT_ID=YourGitHubOAuthClientId
GITHUB_AUTH_CLIENT_SECRET=YourGitHubOAuthClientSecret
./bin/standalone.sh -c standalone-apicurio.xml

Getting Started (Docker)

If you prefer to get started by using the Apicurio Studio docker image, that's cool too! Just follow the instructions in this section instead.

Download

  1. Use docker to pull the Apicurio Studio docker image: apicurio/apicurio-studio 

    docker pull apicurio/apicurio-studio:latest

Configure

  1. In order to log in to the quickstart, you will need to have a GitHub account and you will need to register an OAuth application in GitHub.
  2. Make sure you use the following important settings when you register your OAuth app:
    Homepage URL
    : https://localhost:8443/studio
    Authorization URL
    : https://localhost:8443/studio/callback
  3. Pass environment variables to docker (see below) for your GitHub OAuth app's ClientID and ClientSecret using GITHUB_AUTH_CLIENT_ID and GITHUB_AUTH_CLIENT_SECRET, respectively.

Run

  1. Use docker to run the downloaded image 

    docker run -it -e GITHUB_AUTH_CLIENT_ID=YourOAuthClientId \
    -e GITHUB_AUTH_CLIENT_SECRET=YourOAuthClientSecret \
    -p 8080:8080 -p 8443:8443 apicurio/apicurio-studio:latest

  2. Point your browser at Apicurio Studio and login with your GitHub credentials!

Or try this (after registering an OAuth app)...

docker pull apicurio/apicurio-studio:latest
docker run -it -e GITHUB_AUTH_CLIENT_ID=YourOAuthClientId \
    -e GITHUB_AUTH_CLIENT_SECRET=YourOAuthClientSecret \
    -p 8080:8080 -p 8443:8443 apicurio/apicurio-studio:latest
